## 1. 클린 코드 의의


### 1.1 클린 코드 의의와 필요 이유

실무에서는 클린 코드의 의의를 유지보수 시간의 단축이라고 정의한다.

클린 코드로 인해서 유지보수 시간이 단축된 다는 것은 곧 자원이 절약되는 것이고 자원이 절약되는 것은 돈이 절약되는 것이기 때문에 중요하다.

- 클린 코드가 왜 중요할까?
    
    클린 코드를 작성하게 되면 코드 파악, 디버깅, 리뷰 이 모든 것을 진행하는 시간이 짧이게 된다.  
    그래서 클린 코드라는 개념이 중요하다는 생각이 드는 이유가 결국 AI가 코드를 작성해도 리뷰어가 다시 코드를 리뷰해야 한다.  
    하지만, 내가 기능이 동작하는 지에 대해서는 검토할 수 있는 능력은 있지만 클린 코드인지 판별하는 능력이 없다면 결국 리뷰 부분에서 시간이 지체되며 병목 현상이 발생할 수 있기에 나는 클린 코드가 중요하다고 생각한다.
    

## 2. 안일한 코드 추가


### 2.1 안일한 코드 추가의 함정

아래 코드는 다음과 같은 기능을 한다.

- 질문하기 버튼을 누르면
- 사용자의 약관동의 여부를 확인
- 약관동의가 필요하면 약관동의 팝업열기
- 약관동의 후 질문을 전송, 성공했다는 Alert 띄우기

아래와 같은 요청사항을 반영해서 코드를 수정해야 한다.

- 보험 관련 질문 입력 페이지에서
- 질문하기 버튼을 눌렀을 때
- 사용자에게 배정된 설계사가 있지 확인하고
- 있는 경우, 설계사 사진이 들어간 팝업을 띄우기

아래는 요청사항을 반영한 코드인데, 얼핏 보기엔 그럴듯하고 문제가 없어 보인다.  
하지만, 클린 코드의 관점에서는 나쁜 코드가 되었다.

1. 하나의 목적을 가지고 새롭게 추가된 코드들이 흩어져 있어 한 눈에 들어오지 않고 계속 스크롤을 하여 찾아야 한다.
2. `handleQuestionSubmi` 함수가 여러 일을 할게 되었다.
3. `handleQuestionSubmi`, `handleMyExpertQuestionSubmit` 두 함수 모두 submit 관련 핸들링 함수이지만, 함수의 세부 구현 단계(내부에서 하는 일 정도)가 다르다 보니 코드를 읽을 때 어지럽다.

```jsx
function QuestionPage() {
  const [popupOpened, setPopupOpened] = useState(false); 
  
  async function handleQuestionSubmit() {
    const 연결전문가 = await 연결전문가_받아오기();
    if (연결전문가 !== null) {
      setPopupOpened(true);
    } else {
      const 약관동의 = await 약관동의_받아오기();
      if(!약관동의) {
        await 약관동의_팝업열기();
      }
      await 질문전송(questionValue);
      alert ("질문이 등록되었어요.");
   	}
  }
  async function handleMyExpertQuestionSubmit() {
    await 연결전문가_질문전송(questionValue, 연결전문가.id);
    alert(`${연결전문가.name}에게 질문이 등록되었어요.`);
  }
  return (
    <main>
      <form>
        <textarea placeholder="어떤 내용이 궁금한가요?"/>
        <Button onClick={handleQuestionSubmit}>질문하기</Button>
      </form>
      {popupOpened && (
        <연결전문가팝업 onSubmit={handleMyExpertQuestionSubmit} />
      )}
    </main>
  );
```

  const 약관동의 = await 약관동의_받아오기();

클린 코드는 짧은 코드가 아니고 원하는 로직을 빠르게 찾는 것이고  
원하는 로직을 빠르게 찾기 위해서 아래와 같이 코드가 작성되어야 한다.

- 응집도
    
    하나의 목적을 가진 코드는 흩뿌려져 있으면 안된다.
    
- 단일책임
    
    하나의 함수가 하나의 일만 수행 해야한다.
    
- 추상화
    
    필요한 만큼의 핵심 개념을 노출한다.
    
    이 코드를 핵심 개념만 남기고 추상화를 진행한다면
    
    ```jsx
    <div style={팝업스타일}>
      <button onclick={async () => {
          const res = await 회원가입();
          if(res.success) {
            프로필로 이동();
          }
        }}>전송</button>
    </div>
    ```
    
    아래 코드와 같이 된다.  
    `onSubmit`과 `onSuccess`에서 제출과 성공을 통해 핵심 개념을 알 수 있기 때문에  
    `Popup` 컴포넌트를 사용할 때 컴포넌트의 세부 구현까지 몰라도 된다. 
    
    ```jsx
    <Popup
      onSubmit={회원가입}
      onSuccess={프로필로이동}
    />
    ```

스스로 생각하면서 코드를 작성해야 AI를 이용해서 코드를 작성해도 '왜 이 코드를 이렇게 작성하게 했나요?' 라는 질문을 받아도 자신있게 답변할 수 있을 것 같다.
