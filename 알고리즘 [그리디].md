## 1. 그리디 (Greedy)

### 1.1 그리디 알고리즘이란?

그리디 알고리즘은 흔히 탐욕 알고리즘이라고도 불리며,  
전체 상황을 고려하지 않고 지금 당장 좋아 보이는 선택을 하는 알고리즘이다.  

### 1.2 그리디 알고리즘 특징

1. **핵심**
    1. 나중에 어떻게 되든 상관하지 않고 현재 상황에 나에게 이득이 되는 것을 선택한다.  
2. **장점**
    1. 계산 속도가 매우 빠르고 구현이 정말 간결하다.  
3. **단점**
    1. 항상 최적의 정답을 찾아준다는 보장이 없다.  

### 1.3 그리디 알고리즘이 성립하기 위한 2가지 조건

1. **탐욕적 선택 속성**
    1. 앞의 선택이 이후 선택에 영향을 주지 않아야 한다.  
    즉, 지금의 최선이 미래의 선택지를 망치지 않아야 한다는 확신이 있어야 한다.  
2. **최적 부분 구조**
    1. 문제의 최종 해결 방법은 부분 문제(잘게 쪼개진 문제)에 대한 최적 해결 방법들로 구성된다.  
    즉, 작은 단계에서 모인 최적의 답들이 모여 큰 문제의 정답이 되는 구조다.  

### 1.4 그리디 알고리즘이 실패하는 경우

모든 상황에서 그리디 알고리즘이 통하지 않는다.

ex) 거스름돈 문제에서 동전 단위가 500, 400, 100인 경우에 800을 거슬러줘야 한다면?

- 그리디: 500원(1개) + 100원(3개) = 4개
- 정답: 400원(2개)
- **결론:** 동전 단위들이 서로 배수 관계가 아닐 때는 그리디 알고리즘을 신뢰할 수 없다. 이때는 모든 경우를 다 해보는 완전 탐색이나 DP를 사용해야 한다고 한다.  
  


## 2. 그리디 문제

### 2.1 거스름돈 - 백준 5585 (https://www.acmicpc.net/problem/5585)

- **잘된 점**
    
    처음에는 500, 100, 50원 등 각 동전들을 변수에 하나씩 저장하려고 했는데 너무 비효율적인 것 같고,  
    배열에 담고 for of문을 이용하면 깔끔할 것 같아 for of문을 사용했더니 내 생각대로 쉽게 풀렸다.  
    
- **잘못된 점**
    
    `N`, `n`, `nowNumber` 등 변수 네이밍이 좋지 않아 코드를 이해하는데 쉽지 않다.  
    
    `nowNumber` 라는 불필요한 변수를 사용했다 처음부터 input을 let으로 선언했기에 input 자체에서  
    처리할 수 있지만, 굳이 `nowNumber` 라는 불필요한 변수를 사용한 점이 아쉽다.  
    

```java
const fs = require('fs');
let input = fs.readFileSync("/dev/stdin").toString().trim();

input = 1000 - Number(input)

let nowNumber = input
let N = 0;
let coins = [500, 100, 50, 10, 5, 1]

for(let e of coins){
  let n = Math.floor(nowNumber / e)

  nowNumber -= n * e
  N += n
}

console.log(N)
```

### 2.2 큰 수의 법칙 - 이것이 코딩 테스트다

- **잘된 점**
    
    처음에 계획 한대로 코드를 짜고 조금 수정하니 풀렸다.
    
- **잘못된 점**
    
    맨날 백준에서만 푸니까 콘솔에서 입력 받는 게 어색해서 오류가 많이 났다.
    

```java
const readline = require('readline');

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

rl.question('배열 입력 ex) 2, 3, 4, 5 : ', (input) => {
  rl.question('M K 입력 ex) 8 3: ', (MK) => {

  input = input.split(",").map(Number);
  const [M, K] = MK.split(" ").map(Number);
  let n = 0;
  let res = 0

  input.sort((a, b) => b - a)

  for(let i=0;i<M;i++){
    n++
    if(n > K){
      res += input[1]
      n = 0;
      continue;
    }
    res += input[0]
  }
  console.log(res);
  rl.close();
});
});
```

### 2.3 숫자 카드 게임 - 이것이 코딩 테스트다

- **잘된 점**
    
    문제 이해를 하고 나서는 메서드 몇개 쓰니 바로 풀렸다.
    
- **잘못된 점**
    
    문제 이해 자체를 못해서 문제 이해하는 데 시간을 많이 썼다.  
    구름 코테 이후로 콘솔에서 입력 받는 거 해본 적이 없어서 여기에 시간 많이 쓴게 아쉽다.  
    

```java
const readline = require('readline');

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

let lines = [];

rl.on('line', (line) => {
  lines.push(line);
}).on('close', () => {

  const arr = lines.map(line =>
    line.split(' ').map(Number)
  );

  const res = Math.max(...arr.map(row => Math.min(...row)));

  console.log(res);
});
```

### 2.4 1이 될 때까지 - 이것이 코딩 테스트다

- **잘된 점**
    
    큰 어려움 없이 처음에 생각한 대로 짜고 실행했더니 풀렸다.
    

```java
const readline = require('readline');

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

rl.question('N K 입력 : ', (NK) => {
  let [N, K] = NK.split(" ").map(Number);
  let res = 0

  while(N !== 1){
    if(N % K === 0){
      N /= K
    } else {
      N -= 1
    }
    res++
  }

  console.log(res);
  rl.close();
});
```

무엇보다 알고리즘 이름이 그리디(탐욕)인 이유가 탐욕스러운 사람이 지금 마시멜로 먹는 걸 참으면 미래에 마시멜로 2개를 준다고 해도  
현재 가지고 있는 마시멜로를 그냥 먹어버리는 것처럼 현재의 이익이 중요해서인 게 재밌어서 기억에 남는다.  
  
지금까지는 코테를 많이 풀었어도 어떤 개념을 가지고 풀었다기 보다는 대부분 구현, 정렬 같은 문제다 보니 감으로 푸는 경우가 많았다.  
이제는 방과후 시간에 알고리즘 열심히 배워서 어려운 문제들도 풀어봐야겠다.
