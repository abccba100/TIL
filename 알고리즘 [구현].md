## 1. 구현


### 1.1 구현이란?

문제 설명에 있는 조건이나 상황을 그대로 코드로 옮겨 문제를 푸는 방식이다.

### 1.2 구현 특징

1. **설명이 길다.**
    1. 보통 구현 문제는 로봇, 게임 캐릭터 같은 상황이 붙어 있는 경우가 많다.
2. **조건 많고 복잡하다.**
    1. 이동, 방향, 시간, 날씨 등 다양한 상황과 조건이 등장한다.
3. **문제 이해가 핵심이다.**
    1. 복잡한 알고리즘을 요구하기 보다는 문제를 잘 이해하는 능력을 요구한다.
4. **코드 길이가 길다.**
    1. 복잡한 알고리즘이 들어가지 않더라도 상황, 조건이 많아 짐에 따라 코드 길이가 자연스럽게 길어진다.

### 1.3 구현 문제 풀이 흐름

1. **문제를 천천히 읽는다.**
    1. 문제에서 제공하는 상황과 조건, 입력과 출력을 자세히 보고 이해하는 게 중요하다.
2. **필요한 데이터 구조를 정의한다.**
    1. 변수, 배열, 객체 등 문제 풀이에 사용할 자료형을 선택하는 게 매우 중요하다.
3. **규칙과 조건을 정리한다.**
    1. 문제에서 제공하는 규칙과 조건을 정리하여 의사 코드로 먼저 쓰고 코드를 작성하면 구현 문제 풀이가 쉬워진다.
4. **순서대로 로직을 작성한다.**
    1. 보통은 조건 → 반복 → 상태 변화 순으로 작성된다.

## 2. 구현 문제


### 2.1 시각 - 이것이 코딩 테스트다

- **잘된 점**
    
    처음에 생각한 로직과 코드로 구현하는 부분에서는 큰 문제 없이 잘 작성했다.
    
- **잘못된 점**
    
    처음에는 입력받는 숫자를 number 자료형으로 변환하지 않아서 `N`을 정의할 때 문자열 + 숫자로 결국 문자열이 반환되어 이상한 값이 나왔다.
    

```jsx
const readline = require('readline');

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

rl.question('', (input) => {
  const N = Number(input) + 1;
  let res = 0;

  const isInclude = (n) => {
    const str = String(n)
    return str.includes('3')
  }

  for(let i=0;i<N;i++){
    if(isInclude(i)){
        res += 3600;
        continue;
    }

    for(let j=0;j<60;j++){
      if(isInclude(j)){
        res += 60;
        continue;
      }

      for(let k=0;k<60;k++){
        if(isInclude(k)) res += 1
      }
    }
  }

  console.log(res);
  rl.close();
});
```

### 2.2 상하좌우 - 이것이 코딩 테스트다

- **잘된 점**
    
    쉬워서 큰 문제는 없었다.
    

```jsx
const readline = require('readline');

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

let input = [];

rl.on('line', (line) => {
  input.push(line);
}).on('close', () => {

  const N = Number(input[0]);
  const commands = input[1].split(' ');

  let x = 1;
  let y = 1;

  for (const command of commands) {
    if(command === 'R' && x < N) x++;
    else if(command === 'L' && x > 1) x--;
    else if(command === 'D' && y < N) y++;
    else if(command === 'U' && y > 1) y--;
  }

  console.log(y, x);
});
```

### 2.3 왕실의 나이트 - 이것이 코딩 테스트다

- **잘된 점**
    
    `steps`로 경우의 수를 묶어 처리하여 코드가 깔끔해졌다.
    
- **잘못된 점**
    
    처음에 여러 경우를 처리하는 게 헷갈려서 이상하게 작성됐다.
    

```jsx
const readline = require('readline');

const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

rl.question('', (input) => {

  let now = input.trim()

  let [y, x] = now.split("")
  x = parseInt(x)
  y = y.charCodeAt() - 96;

  let res = 0;

  steps = [[-2, -1], [-1, -2], [1, -2], [2, -1], [2, 1], [1, 2], [-1, 2], [-2, 1]]

  for (const step of steps) {
    if((x + step[0] >= 1 && x + step[0] <= 8 && y + step[1] >= 1 && y + step[1] <= 8)) res++;
  }

  console.log(res)
  rl.close();
});
```

이제는 이게 특정 알고리즘을 이용해야 하는 건지 단순 구현 문제인지 구분하고 풀 수 있을 것 같아 코테에 더 자신감이 붙는다.
