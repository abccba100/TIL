## 1. 실행 컨텍스트


### 1.1 실행 컨텍스트란?

JS엔진이 코드를 실행할 때 어떤 변수가 선언되어 있는지, `this` 는 무엇을 가리키는지, 외부 스코프에 대한 정보 등을 필요로 한다.  
이 정보를 하나로 묶어놓은 객체가 실행 컨텍스트이다.

### 1.2 실행 컨텍스트와 콜 스택 관계

콜 스택은 함수의 호출을 기록하고 추적하기 위해 관리한다.  
쉽게 정리하면 콜 스택은 함수를 실행을 관리한다. 이 때 함수 안에 코드를 실행 시키려면 위에서 말했 듯이
여러 정보가 필요하다.  
때문에 함수를 실행하게 되면 콜 스택에는 해당 함수에 대한 실행 컨텍스트가 쌓이게 되는 것이다.

ex) A라는 함수가 있다고 가정한다면,  
A 함수를 실행시키면 콜 스택에 A 함수의 실행 컨텍스트가 쌓이게 된다.  
그리고 A 함수의 실행이 종료되면 맨위 있는 A 함수의 실행 컨텍스트는 제거된다.

### 1.3 변수 환경 (Variable Environment)

변수 환경은 아래와 같은 정보들을 담는다.

`var` 선언, 함수 선언  
실행 컨텍스트는 코드 실행 전 생성 단계에서 `var` 는 undefind로,  
함수 선언은 함수 전체로 등록되기 때문에 선언 전에 접근해도 에러가 나지 않는 것이다.  
이게 호이스팅의 원리다.

- 전역 컨텍스트의 경우는 어떨까?
    
    전역 컨텍스트에는 어떤 함수 안에 선언된 변수, 함수들이 아닌 전역 변수나, 전역 함수로 선언된 정보들이 담기게 된다.
    

### 1.4 렉시컬 환경 (Lexical Environment)

렉시컬 환경은 아래와 같은 정보들을 담는다.

`let` /`const` , 외부 스코프
외부 스코프를 가져와 컨텍스트들을 체인으로 연결하여 변수 탐색 경로를 만들기 때문에 함수가 종료된 후에도 환경을 붙잡고 있을 수 있어서 클로저가 동작할 수 있다.

- 왜 `var` 와 `let` /`const` 스코프가 다를까?
    
    원래 `var` 는 `{ }` 이런 블록을 만나도 변수가 블록 안에서 다시 생성되지 않는다.  
    그렇기 때문에 function scope를 가지게 되는 것이고  
    `let` /`const` 는 `{ }` 이런 블록을 만나면 새 Lexical Environment가 생성되고,  
    그 블록 안의 let/const는 새 환경에 등록된다  
    그렇기 때문에 block scope를 가지게 되는 것이다.
    

### 1.5 This Binding

This Binding은 앞서 말한 두 환경과 성격이 조금 다르다.  
특히 렉시컬 환경에 경우에는 함수가 정의된 스코프를 기준으로 상위 스코프를 가져오지만,  
This Binding은 어떻게 호출 되었는지에 따라 결과가 달라진다.

화살표 함수는 This Binding을 만들지 않고 상위 컨텍스트의 This Binding을 사용하기 때문에  
우리가 아는 화살표 함수와 `this` 의 관계가 만들어지는 것이다.

---

실행 컨텍스트 개념 자체에 클로저, this와 화살표 함수의 관계, var와 let/const, 호이스팅, function scpoe와 block scope 등 많은 개념들이 얽혀 있어서  
생각보다 재밌는 개념 공부였고, 콜 스택 자체에 대한 이해도가 함께 올라간 느낌이라 개념 공부 하나에서 얻어가는 것이 많은 것 같다.
