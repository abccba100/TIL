## 1. 디바운싱


### 1.1 디바운싱 개념

디바운싱은 연이어 발생하는 이벤트들을 처음이나 마지막에 한 번만 실행하는 것을 의미한다.

디바운싱이라는 게 왜 중요할까?  
연이어 발생하는 이벤트를 한 번만 실행한다면 서버에 주는 부담도 훨씬 줄일 수 있어서 매우 의미있는 최적화 방법이라고 생각한다.  

### 1.2 디바운싱 활용

대표적인 디바운싱 활용 예시로는 검색이 있다.

```jsx
  const debounceTime = 150; /* 디바운스 시간 */

  useEffect(() => {
    const debounceTimer = setTimeout(() => {
      setDebouncedValue(value);
    }, debounceTime);

    return () => clearTimeout(debounceTimer);
  }, [value]);
```

위와 같이 코드를 작성하면 새로운 글자가 들어와 value가 변하고 아무런 변화가 없다면 150ms 뒤에 api 호출을 하게 된다.  
하지만, 150ms가 지나기 전에 여러 글자를 계속해서 입력하면 그 때를 기준으로 다시 150ms가 지난 뒤에 한 번만 api 호출을 하게 되면서 서버에 부담을 줄일 수 있다.

### 1.3 디바운싱 문제

디바운싱을 사용하면 서버에 주는 부담을 줄일 수 있지만 디바운스 타임(디바운싱 적용되는 시간) 만큼 API 요청이 늦게 날아가게 된다.

즉, 사용자가 검색어를 입력하여도 위 상황에서는 150ms 늦게 API 요청이 가게 되어 UX가 떨어지게 되는 문제가 생긴다.

### 1.4 디바운싱 문제 해결

많은 방법들이 있겠지만, 내가 느끼기에 가장 효과적인 방법은 글자 수에 따라 디바운스 타임을 다르게 주는 것이다.

글자 수가 적을 때는 아직 검색어가 완성되지 않았을 확률이 높기 때문에 300ms 처럼 높은 디바운스 타임을 주고, 글자 수가 많을 때는 검색어가 완성되었을 확률이 높기 때문에 150ms 처럼 낮은 디바운스 타임을 줘서 최대학 UX 하락을 방지한다.

그 외에도 loading UI나 prefetch 같은 것이 있겠지만, 이런 건 UX를 위해서 기본적으로 적용되는 것이기 때문에 제외함

## 2. 쓰로틀링


### 2.1 쓰로틀링 개념

쓰로틀링은 연이어 발생하는 이벤트를 일정시간 마다 한 번씩만 실행하게 하는 것을 의미한다.

디바운싱이 이벤트를 마지막에 한 번만 실행하는 것에 가까웠다면,  
쓰로틀링은 주기적으로 실행을 제한하는 것에 가깝다.

쓰로틀링이라는 게 왜 중요할까?

스크롤, 마우스 이동 같은 이벤트는 매우 짧은 시간 동안 엄청나게 많이 발생한다.  
이 때 이벤트마다 API 호출이나 무거운 연산을 실행하면 성능 저하가 발생할 수 있기 때문에,  
일정 시간 간격으로만 실행되게 제한하여 성능을 최적화한다.

### 2.2 쓰로틀링 활용

대표적인 쓰로틀링 활용 예시로는 스크롤 이벤트가 있다.

```jsx
const throttleTime = 300;

let waiting = false;

window.addEventListener("scroll", () => {
  if (waiting) return;

  waiting = true;

  console.log("스크롤 위치 계산");

  setTimeout(() => {
    waiting = false;
  }, throttleTime);
});
```

위와 같이 코드를 작성하면 스크롤 이벤트가 계속 발생하더라도 300ms 마다 최대 1번만 이벤트가 발생한다.

이를 통해서 불필요하게 과한 연산 횟수를 줄여 렌더링 성능을 증가 시키거나 서버에 부담을 줄일 수 있다.

---
서버 가해지는 부담을 줄이는 최적화지만, 생각보다 꽤 큰 영향을 끼쳐서 이런 최적화 관련된 공부를 계속 해봐도 괜찮을 것 같다.
또, 디바운싱을 쓰면 검색 때 발생하는 UX 하락 문제도 어떻게 해결하면 좋을지 생각해봐야 할 것 같다.
