## 1. 자료구조


### 1.1 자료구조란?

코딩 테스트에서는 문제를 해결할 때 데이터의 저장 및 처리를 빠르게 처리할 때 사용하는 구조를 말한다.

### 1.2 자료구조 특징

1. **문제 유형에 따라 쓰는 자료구조가 정해진다.**
    1. 큐, 스택, 해시 등 많은 자료구조 중 어떤 것을 쓸지 정해야 한다.
2. **시간 복잡도가 중요하다.**
    1. 자료구조를 쓰는 이유 중 하나가 빠른 데이터의 처리이기 때문에 주로 시간 복잡도가 중요하다.

### 1.3 자료구조 문제 풀이 흐름

1. **문제 유형 파악를 파악한다.**
    1. 탐색인지, 정렬인지, 그래프인지 구분
2. **자료구조 선택를 선택한다.**
    1. 삽입/삭제 → 스택/큐
    2. 중복 체크 → Set
    3. 위와 같이 문제 유형에 따라 올바른 자료 구조를 선택한다.
3. **데이터의 흐름을 정리한다.**
    1. 의사 코드를 작성하고 데이터를 어떻게 처리할지 정리한다.

## 2. 자료구조 문제


### 2.1 같은 숫자는 싫어 - 프로그래머스

- **잘된 점**
    
    쉬운 문제라 큰 문제 없이 풀렸다.
    
- **잘못된 점**
    
    자료구조를 활용해서 풀어야 한다고 들었는데 그냥 for문과 push로 풀었다.
    

```jsx
function solution(arr)
{
    let answer = [];
    let nowNumber = arr[0];

    for(let i=1; i < arr.length + 1; i++){
      if(nowNumber !== arr[i]){
        answer.push(nowNumber);
        nowNumber = arr[i];
      }
    }
    
    return answer;
}
```



### 2.2 올바른 괄호 -  프로그래머스

- **잘된 점**
    
    처음에는 막혔지만, 의사 코드 작성하면서 푸니 크게 어렵지는 않았다.
    
- **잘못된 점**
    
    예전에 풀었던 문제라 의사 코드 없이 바로 구현하려고 하니 생각보다 쉽게 막혔다.
    

```jsx
function solution(s){
  var answer = true ;
  let stack = [];
  for(let i=0;i<s.length;i++){
    if(s[i] === '('){
      stack.push(s[i]);
    }
    else{
      if(stack.length === 0){
        answer = false;
        break;
      }
      stack.pop();
    }
  }
  if(stack.length>0){
    answer = false;
  }

  return answer;
}
```

### 2.3 기능 개발 - 프로그래머스

- **잘된 점**
    
    막힌 부분에서 의사 코드 작성하며  풀었고, while 이용해 가독성 좋게 작성한 것 같다.
    
- **잘못된 점**
    
    크게 잘못된 점은 없지만, 코드를 더 간단하게 줄일 수 있을 것 같은 느낌이 든다.
    

```jsx
function solution(progresses, speeds) {
  let answer = [];
  let count = 0
  
  while(progresses.length !== 0){

    for(let j=0;j<progresses.length;j++){
      progresses[j] += speeds[j];
    }

    while(progresses[0] >= 100){
      progresses.shift()
      speeds.shift()
      count++
    }

    answer.push(count)
    count = 0
  }

  answer = answer.filter(e => e !== 0)
  return answer;
}
```

### 2.4 프로세스 - 프로그래머스

- **잘된 점**
    
    문제 이해한 후에는 막힘 없이 바로 작성됐다.
    
- **잘못된 점**
    
    처음에 문제 이해를 못해서 ‘왜 결과 값이 이렇게 나오지?’ 하면서 시간을 많이 태웠다.
    

```jsx
function solution(priorities, location) {
  var answer = 0;
  let temp;

  
  while(priorities.length !== 0){

    if(priorities[0] < Math.max(...priorities)){
      temp = priorities.shift()
      priorities.push(temp)
      if(location === 0){
        location = priorities.length - 1
      } else{
        location--
      }
    } else{
      answer++
      priorities.shift()
      if(location === 0){
        break
      } else{
        location--
      }
    }
  }
  return answer;
}
```

### 2.5 명예의 전당 - 프로그래머스

- **잘된 점**
    
    비교, 정렬 문제라서 딱히 어렵지 않았다.
    
- **잘못된 점**
    
    가독성이 안좋다 지금 코드에서도 else if{}를 안쓰고 else{} 안에 if{} 만 쓰고 있는 것처럼 개선할 수 있는 부분이 많을 것 같다.
    

```jsx
function solution(k, score) {
  var answer = [];
  let tops = []

  for(let i=0;i<score.length;i++){
    if(tops.length < k){
      tops.push(score[i])
      tops.sort((a, b) => b - a)
      answer.push(tops[tops.length - 1])
    } else{
      if(tops[k - 1] < score[i]){
        tops[k - 1] = score[i]
        tops.sort((a, b) => b - a)
      }
      answer.push(tops[k - 1])
    }
  }
  return answer;
}
```
